# Paper Trading System -- สรุปแบบเข้าใจง่าย

## แนวคิดหลัก

ใช้ **BTC เป็นตัวนำ** (BTC-Led Strategy) -- วิเคราะห์สภาพตลาด BTC จาก 8 ปัจจัย แล้วสร้าง "คะแนนรวม" (Composite Score)
ถ้าคะแนนบวกมากพอ = ตลาดน่าจะขึ้น → เปิด LONG ทั้ง 6 เหรียญ
ถ้าคะแนนลบมากพอ = ตลาดน่าจะลง → เปิด SHORT ทั้ง 6 เหรียญ

**ไม่ได้วิเคราะห์แต่ละเหรียญแยก** -- ใช้สัญญาณจาก BTC อย่างเดียว แต่เทรดหลายเหรียญเพื่อกระจายความเสี่ยง

---

## เหรียญที่เทรด

| เหรียญ | Threshold | Cooldown |
|--------|-----------|----------|
| BTC | ±2.5 | 4 bars (1 ชม.) |
| XRP | ±3.5 | 4 bars |
| ADA | ±3.5 | 4 bars |
| DOT | ±3.0 | 8 bars (2 ชม.) |
| SUI | ±3.0 | 4 bars |
| FIL | ±3.0 | 4 bars |

- Threshold สูง = ต้องมั่นใจมากกว่าถึงจะเข้า (XRP, ADA ใช้ 3.5 เพราะต้องการสัญญาณที่แรงกว่า)
- Threshold ต่ำ = เข้าง่ายกว่า (BTC ใช้ 2.5)

## 8 ปัจจัยที่ใช้คำนวณคะแนน (v3 Model)

คะแนนรวม = ผลรวมของ 8 ปัจจัย แต่ละตัวให้คะแนน +/- ตาม weight

### ปัจจัยเดิม (5 ตัว)

| # | ปัจจัย | Weight | หลักการ | ตัวอย่าง |
|---|--------|--------|---------|----------|
| 1 | **Liquidation Cascades** | 2.0 | ถ้ามีคนโดนบังคับปิด SHORT เยอะผิดปกติ = ราคาน่าจะพุ่ง (short squeeze) | Short liq พุ่ง 3 เท่า → +2.0 คะแนน |
| 2 | **Funding Rate** | 2.0 | ถ้าคนจ่ายค่า funding ติดลบ = คนถือ SHORT เยอะ = contrarian bullish | FR < -0.01% → +2.0 คะแนน |
| 3 | **ETF Flows** | 1.0 | เงินไหลเข้า BTC ETF = สถาบันซื้อ = bullish | ETF flow MA > $50M → +1.0 คะแนน |
| 4 | **Whale Alerts** | 1.5 | วาฬโอนเข้า exchange = อาจขาย = bearish, โอนออก = bullish | Net whale > $50M → +1.5 คะแนน |
| 5 | **OI Divergence** | 0.5 | ราคาขึ้น + OI ขึ้น = bullish, ราคาลง + OI ขึ้น = bearish divergence | Price up + OI up → +0.25 คะแนน |

### ปัจจัยใหม่ v3 (3 ตัว)

| # | ปัจจัย | Weight | หลักการ | ตัวอย่าง |
|---|--------|--------|---------|----------|
| 6 | **Order Book Combined** | 2.0 | **Contrarian** -- ถ้า bid เยอะกว่า ask ผิดปกติ = คนรอซื้อเยอะ = อาจกลับตัวลง | Bid >> Ask → -2.0 คะแนน (bearish) |
| 7 | **Basis Contrarian** | 1.5 | **Contrarian** -- futures premium สูงมาก = คนมั่นใจเกินไป = อาจกลับตัว | Basis z-score > 1.5 → -1.5 คะแนน |
| 8 | **Tick Liquidation** | 2.0 | เหมือน #1 แต่ใช้ข้อมูลระดับ tick (ละเอียดกว่า, real-time จาก WebSocket) | Short liq net > 2 → +2.0 คะแนน |

### ตัวอย่างการคำนวณ

```
สมมติสภาพตลาด:
  - Short squeeze เกิดขึ้น (Liquidation)     → +2.0
  - Funding rate ติดลบ                        → +2.0
  - ETF เงินไหลเข้า                          → +1.0
  - Whale โอนออก exchange                     → +1.5
  - OI divergence bullish                     → +0.25
  - Order book: bid มากกว่า (contrarian)      → -2.0
  - Basis ปกติ                                → 0.0
  - Tick liq: short โดน liq เยอะ             → +2.0
  ─────────────────────────────────────────────
  คะแนนรวม = +6.75

  BTC threshold = 2.5 → เข้า LONG ✓
  XRP threshold = 3.5 → เข้า LONG ✓
  ADA threshold = 3.5 → เข้า LONG ✓
```

## เงื่อนไขเข้าเทรด

ต้องผ่านทุกข้อถึงจะเปิด position:

1. **คะแนนเกิน threshold** -- BTC score ≥ +threshold (LONG) หรือ ≤ -threshold (SHORT)
2. **ไม่มี position เปิดอยู่** -- เหรียญนั้นต้องว่างอยู่ (เปิดได้แค่ 1 position ต่อเหรียญ)
3. **ผ่าน cooldown** -- หลังปิด position ต้องรอ N bars ก่อนเข้าใหม่ (ป้องกัน overtrade)
4. **PA filter ปิดอยู่** -- ตอนนี้ทุกเหรียญ `use_alt_pa_filter = False` (ไม่ใช้ EMA alignment)

### รายละเอียดการเปิด

- **ขนาด**: $1,000 × 2x leverage = $2,000 notional ต่อ position
- **Entry price**: ราคาปิดของแท่งเทียนล่าสุด + slippage 0.015%
- **ค่าธรรมเนียม**: 0.02% ต่อทาง (เข้า + ออก)

## เงื่อนไขออกจากเทรด (เรียงตามลำดับความสำคัญ)

ทุกรอบ 15 นาที ระบบจะเช็คตามลำดับนี้ -- แตะตัวไหนก่อนปิดเลย:

| ลำดับ | เงื่อนไข | วิธีคำนวณ | ตัวอย่าง BTC (ATR ~$1,500) |
|-------|----------|-----------|---------------------------|
| 1 | **Stop Loss (SL)** | Entry ± 2.5 × ATR | LONG entry $85,000 → SL = $81,250 |
| 2 | **Trailing Stop** | ปิดอยู่ (mult=99) | ไม่ใช้งานในตอนนี้ |
| 3 | **Take Profit (TP)** | Entry ± 4.0 × ATR | LONG entry $85,000 → TP = $91,000 |
| 4 | **Timeout** | ถือครบ 96 bars (24 ชม.) | ปิดที่ราคาปิดของแท่งเทียน |
| 5 | **Signal Flip** | คะแนนกลับทิศ | ถือ LONG อยู่แล้วได้สัญญาณ SHORT → ปิด LONG ที่ราคาเปิดของแท่งเทียน |

### Risk/Reward

- **SL**: 2.5 ATR = ขาดทุนสูงสุดประมาณ $10-12 ต่อ position (ขึ้นกับเหรียญ)
- **TP**: 4.0 ATR = กำไรเป้าหมายประมาณ $16-24 ต่อ position
- **Risk:Reward ≈ 1:1.6**

## ข้อสังเกตสำคัญ

1. **Contrarian เป็นหลัก** -- หลายปัจจัยใช้แบบ "สวนกระแส" (Order Book, Basis, Liquidation) เช่น คนรอซื้อเยอะ → ระบบมองว่า bearish
2. **SHORT ดีกว่า LONG** -- จาก backtest สัญญาณ SHORT มี win rate 60-68% vs LONG 37-46%
3. **ทุกเหรียญใช้สัญญาณเดียวกัน** -- ต่างแค่ threshold กับ cooldown
4. **ไม่ใช้ Fear & Greed, Taker Ratio, LS Ratio** -- เคยทดสอบแล้วพบว่าทำให้แย่ลง ถูกถอดออกใน v3
5. **Backtest OOS: +$18,056 (180% ของทุน $10K)** ใน 3 เดือน -- แต่ผล paper trade จริงต้องรอดู
